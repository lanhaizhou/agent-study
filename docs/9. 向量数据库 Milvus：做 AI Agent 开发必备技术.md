# 向量数据库 Milvus：做 AI Agent 开发必备技术 - 总结

对应代码：课程仓库 [ai-agent-course-code](https://github.com/QuarkGluonPlasma/ai-agent-course-code) 内 Milvus 相关示例（如 `src/insert.mjs`、`src/query.mjs`、`src/rag.mjs`、`src/update.mjs`、`src/delete.mjs`）。环境变量与第 6、7 章一致：根目录 `.env` 配置 `OPENAI_API_KEY`、`OPENAI_BASE_URL`、`MODEL_NAME`、`EMBEDDINGS_MODEL_NAME`；依赖需包含 `@zilliz/milvus2-sdk-node`、`@langchain/openai`（ChatOpenAI、OpenAIEmbeddings）、`dotenv`。本地需用 Docker 运行 Milvus（见下文）。

---

## 一、为什么需要 Milvus

- 第 6、7 章用 **MemoryVectorStore** 在内存中存向量，适合演示；实际 AI Agent 产品通常用 **Milvus** 等持久化向量数据库，便于扩容、多实例共享、知识/记忆长期存储。
- 与关系型数据库类比：Web 应用用 MySQL 做增删改查、多表关联；AI Agent 用 Milvus 存知识/记忆，基于**向量相似度检索**和增删改实现 RAG、记忆召回等功能。二者常**双写**：MySQL 负责按 id、关键词的精确查询与业务表关联，Milvus 负责按自然语言的语义检索。

## 二、Milvus 与 MySQL 的分工

- **MySQL**：按 id、关键词匹配查询，适合「列表、详情、条件筛选」等精确查询；无法做「意思相近」的语义检索。
- **Milvus**：按**向量相似度**（如余弦相似度）检索，支持用自然语言 query 向量化后查最相关的文档/记忆；适合「我哪几天心情比较好」「和某主题相关的日记」等语义类查询。
- **典型用法**：同一份业务数据（如日记）同时写入 MySQL 与 Milvus；查列表、按日期/标签筛选用 MySQL；按自然语言问「心情」「做了什么」等走 Milvus 检索再交给大模型回答。

## 三、本地运行 Milvus 与 Attu

- **Docker**：需先安装 [Docker](https://www.docker.com/)；从 [Milvus releases](https://github.com/milvus-io/milvus/releases) 下载 `milvus-standalone-docker-compose.yml`，在放配置的目录执行：`docker compose -f ./milvus-standalone-docker-compose.yml up -d`。
- **端口**：Milvus 服务默认 **19530**；健康检查可访问 `http://localhost:9091/healthz`。
- **Attu**：Milvus 的 [Attu](https://github.com/zilliztech/attu) GUI，用默认配置连接 `localhost:19530` 即可查看 database、collection、entity，便于调试与查看插入/删除结果。

## 四、Milvus 数据结构与索引

- **层级**：**database → collection → entity**；每个 collection 对应一张「表」，每条数据为一个 entity。
- **Schema**：创建 collection 时需定义字段，例如：`id`（主键，VarChar）、`vector`（FloatVector，维度需与嵌入模型一致，如 1024）、`content`、`date`、`mood`、`tags`（Array 等）等；与 MySQL 表结构类似，核心区别是 **vector** 字段用于相似度检索。
- **索引**：对 **vector** 字段建索引（如 **IVF_FLAT** + **COSINE** 余弦相似度），才能高效做向量搜索；创建索引后需 **loadCollection** 才能查询。

## 五、增删改查与代码要点

| 操作     | 说明                                                                                                                                                                                                                                        |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **插入** | **createCollection** 指定 schema（含 id、vector、业务字段）→ **createIndex** 对 vector 建索引（IndexType、MetricType、params）→ **loadCollection** → **insert** 写入 data；vector 需用**嵌入模型**对 content 等文本做 **embedQuery** 得到。 |
| **查询** | 将用户 query 用嵌入模型 **embedQuery** 得到向量 → **client.search** 指定 collection_name、vector、limit、metric_type（COSINE）、output_fields，返回最相似的若干条 entity 及 score。                                                         |
| **更新** | Milvus 通过 **upsert** 实现：传入同 id 的完整 entity（含重新向量化后的 vector），会按 id 覆盖；更新涉及文本内容时需重新调用嵌入模型生成 vector。                                                                                            |
| **删除** | **client.delete** 指定 collection_name 和 **filter** 表达式：如 `id == "xxx"` 按 id 删、`id in ["a","b"]` 批量删、`mood == "sad"` 按条件删；**不需要**嵌入模型。                                                                            |

- Node 连接：`new MilvusClient({ address: 'localhost:19530' })`，使用前可 `await client.connectPromise`。嵌入模型需与建表时 vector 维度一致（如 **OpenAIEmbeddings** 的 `dimensions: 1024`）。

## 六、接入 RAG 流程与依赖环境

- **RAG 流程（rag.mjs）**：用户问题 → 用嵌入模型将问题向量化 → 在 Milvus 中 **search** 取最相关 k 条（如日记）→ 将检索到的 content、date、mood 等拼成上下文 → 写入 prompt（系统提示 + 上下文 + 用户问题）→ 调用 **ChatOpenAI** 生成回答。与第 6 章「向量库检索 + 增强 prompt + 生成」一致，仅把 **MemoryVectorStore** 换成 **Milvus**；检索接口为 `client.search`，返回的 results 带 score 与 output_fields，按需拼接成字符串放入 prompt 即可。
- **依赖**：`@zilliz/milvus2-sdk-node`、`@langchain/openai`、`dotenv`。**环境变量**：与第 6、7 章相同。**运行顺序**：先 Docker 启动 Milvus，再执行 insert（建表、插数据），之后可跑 query、rag、update、delete。

## 七、总结

- **Milvus**：持久化向量数据库，用于知识、记忆等语义检索；与 MySQL 分工明确：MySQL 做 id/关键词精确查询与业务关联，Milvus 做自然语言语义检索；实践中常对同一业务数据做 **MySQL + Milvus 双写**。
- **本地**：Docker + milvus-standalone-docker-compose 启动；端口 19530；可用 Attu 做 GUI 查看与调试。
- **数据与索引**：collection 含 schema（id、vector、业务字段）；vector 字段需建索引（如 IVF_FLAT + COSINE）并 loadCollection 后才能检索。
- **增删改查**：插入/更新需嵌入模型生成 vector；查询为 query 向量化后 search；删除用 filter，不需嵌入。接入 RAG 时：Milvus 检索 → 拼上下文 → prompt → 大模型生成，与第 6 章流程一致。
- **扩展**：做 AI Agent 项目时，Milvus 是必备能力，可支撑知识库、用户记忆、语义检索等；简历与面试可围绕「向量库选型、双写、RAG 链路」展开。
