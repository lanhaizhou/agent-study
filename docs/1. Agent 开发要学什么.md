# AI Agent 开发要学什么 - 总结

## 一、为什么不能只「调大模型」

大模型本身有局限，需要由开发者做扩展：

- **记不住**：历史对话记不住 → 需要 **Memory** 记忆管理
- **做不了**：只能给思路，不能真正操作（如访问网页）→ 需要 **Tool** 工具调用
- **不知道**：私密文档、内部知识、最新新闻 → 需要 **RAG** 文档/知识库查询

大模型无法自行具备这些能力，需要开发者在系统侧做 Memory、Tool、RAG 等扩展。

## 二、Agent 是什么

**Agent = 大模型 + Tool + Memory（+ RAG）**

核心是给大模型加上 **Tool** 和 **Memory**；RAG 是常见的扩展能力之一。

- 大模型负责思考、规划
- **Tool**：扩展「能做」的事（读写文件、执行命令、操作浏览器等）
- **Memory**：管理「该记住」的信息
- **RAG**：查询内部知识库，补充大模型不知道的内容

具备这些能力、能帮你完成具体任务的，就是一个 Agent。Cursor、Manus、支付宝理财助手等，都是 Agent 产品。

## 三、常见产品是怎么实现的

- **Cursor**：读写文件、执行命令 → 通过 **Tool** 扩展
- **Manus**：打开浏览器、访问网页、点击元素、写 md → 通过 **Tool** 扩展
- **支付宝理财助手**：查基金数据、做推荐 → 通过 **RAG** 访问内部知识库

光会和大模型聊天不够，要会扩展 Tool、Memory、RAG，才能满足真实业务需求。

## 四、用什么框架学

- **LangChain**：单 Agent 开发，对 Memory、Tool、RAG 等封装了 API，直接调用即可
- **LangGraph**：基于 LangChain，专门做**多 Agent 协作**，解决多 Agent 之间的交互问题

课程技术栈：**Node + LangChain + LangGraph**。逻辑、概念与 Python 版一致，学会后换语言也容易。

## 五、为什么要学「AI 全栈」

- AI 逻辑通常跑在后端：记忆存 Redis、知识存 MySQL 等，需要后端能力
- 只有把 AI 和后端结合起来，才能做出可落地的产品

因此课程是 **后端 + AI** 一起学：Node 后端用 **Nest**，配合 LangChain/LangGraph 做 AI 全栈产品。

## 六、总结

- **学什么**：给大模型扩展 **Tool、Memory、RAG**，开发能真正干活的 Agent
- **用什么学**：**LangChain**（单 Agent）+ **LangGraph**（多 Agent），Node 版
- **怎么学**：结合 **Nest 后端**，做 AI 全栈，让 AI 落地成产品、变成自己的竞争力
